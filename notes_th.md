เรากำลังเรียนรู้เพื่อพัฒนาเป็นนักพัฒนา JavaScript

JavaScript เป็นภาษาโปรแกรมที่ใช้สร้างเว็บแอปพลิเคชันสมัยใหม่แบบ full-stack

เว็บแอปพลิเคชันแบบสมบูรณ์ประกอบด้วยระบบ frontend, backend และฐานข้อมูล การพัฒนาระบบเหล่านี้ต้องอาศัยทั้งการประสานงานของระบบภายในโปรเจกต์ และการเชื่อมต่อกับระบบภายนอก เช่น ระบบชำระเงิน ผู้ให้บริการยืนยันตัวตน cloud storage บริการอีเมล และ third-party APIs

สำหรับผู้เรียนจำนวนมาก การเข้าใจแนวคิดพื้นฐานของ OOP ใน JavaScript เช่น constructor, class และ object instance จะช่วยสร้างรากฐานที่แข็งแรงสำหรับการทำความเข้าใจ Promises และเมื่อเข้าใจ Promises แล้ว ก็จะช่วยให้เข้าใจ JavaScript แบบ asynchronous ได้ง่ายขึ้น

แม้ว่าแนวทางการทำงานในอุตสาหกรรมปัจจุบันอาจให้ความสำคัญกับการเขียนแบบ function-first มากกว่า แต่ผู้เริ่มต้นก็ยังได้ประโยชน์จากการเข้าใจเรื่อง object, instance และพื้นฐานของ OOP เพราะตัวภาษา JavaScript เอง รวมถึง APIs ที่มีมาในตัวจำนวนมาก ก็ยังอาศัยแนวคิดแบบ object-based อยู่

Programming paradigms เป็นแนวทางที่ช่วยจัดระเบียบว่าเราจะเขียนโค้ดและวางโครงสร้างตรรกะของโปรแกรมในเชิงแนวคิดอย่างไร ส่วน folder structure คือการจัดระเบียบว่าไฟล์ต่าง ๆ จะถูกวางไว้ตรงไหนในโปรเจกต์จริง

เราสามารถเปรียบเทียบได้ดังนี้:

- **OOP / Functional programming** = วิธีที่เราใช้คิดและจัดโครงสร้างตรรกะของโปรแกรม
- **Folder structure** = วิธีที่เราจัดวางไฟล์ภายใน codebase
